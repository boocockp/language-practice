Technical feature: Verb conjugation
===================================

Overview
--------

The first implementation of the verb Handlebars helper for French fails in Convex because it loads a very large dictionary of conjugations that will not fit in the available memory.  Unit tests run locally with ample memory work correctly.  We therefore need to find a way of generating verb conjugations without using such a large data structure.  The approach will be to use simple rules for regular verbs, and to take advantage of common patterns among irregular verbs.

Preparation
-----------

- Find one or more sources that explain the _rules_ and _patterns_ for conjugating French verbs, not just simple lists of conjugations.

Approach
--------

- Continue to use nlg.verbsManager.getAgreeVerb in the helper
- When the NlgLib is created, call nlg.verbsManager.setEmbeddedVerbs with a replacement verb lookup
- The new verb lookup is a Proxy that acts like the VerbsInfo interface - which is just a dictionary of verb => conjugation table (see below)
- The verb lookup will contain a dictionary of irregular verbs only
- Implement the get of the Proxy to return the dictionary entry if it exists, otherwise create the conjugation table using the regular verb rules
- Create the regular verb rules for -er, -ir and -re verbs by using a French grammar rule source and the conjugation table format below
- Create the irregular verb dictionary and the regular verb rules in separate files.  The irregular verb dictionary is initially empty
- Use the full original dictionary of conjugations in french-verbs-lefff/dist/conjugations.json to generate the irregular verb dictionary, using the following steps:
    - Create a one-off script in scripts folder that takes every verb in conjugations.json and calls the regular verb rules with it
    - If the conjugation table generated by the rules does not match the one in conjugations.json, add the entry from conjugations.json to the irregular verb dictionary
- Finally, create a test that compares every entry in conjugations.json with the output of the proxy and verifies that they match    


Conjugation table format
------------------------
- The conjugation table corresponds to the VerbInfo interface
- The VerbInfo interface is a map of code letters to an array of conjugations or participles
- The meaning of the code letters are given in this document: https://github.com/RosaeNLG/rosaenlg/blob/master/packages/french-verbs-lefff/resources/lefff-tagset-0.1.2.pdf
- The G entry has just one form of the verb - the present participle
- The W entry has just one form of the verb - the infinitive
- The K entry has the forms of the past participle agreed with masculine singular, masculine plural, feminine singular, feminine plural
- The other entries are the forms for 1st, 2nd and 3rd person, singular and plural
- The Y entry (imperative) has "NA" for forms that do not exist
- Example entry for regular verb 'parler':
`"parler": {
        "P": [ "parle", "parles", "parle", "parlons", "parlez", "parlent" ],
