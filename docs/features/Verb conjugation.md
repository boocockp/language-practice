Technical feature: Verb conjugation
===================================

Overview
--------

The first implementation of the verb Handlebars helper for French fails in Convex because it loads a very large dictionary of conjugations that will not fit in the available memory.  Unit tests run locally with ample memory work correctly.  We therefore need to find a way of generating verb conjugations without using such a large data structure.  The approach is to use simple rules for regular verbs, and an irregular-verb dictionary for the rest.

Implementation (current)
------------------------

- **Verb lookup**: When `NlgLib` is created for French (`fr_FR`), `nlg.verbsManager.setEmbeddedVerbs(createVerbLookupProxy(irregularVerbs))` is called so that all conjugation lookups use our Proxy instead of the default dictionary.
- **Proxy** (`convex/frenchConjugation/verbLookupProxy.ts`): Implements the VerbsInfo interface. On `proxy[verb]`, returns the irregular dictionary entry if present, otherwise the conjugation table from regular verb rules (-er, -ir, -re).
- **Regular verb rules** (`convex/frenchConjugation/regularVerbRules.ts`): Pure functions that, given an infinitive, return a full VerbInfo for -er (parler type), -ir (finir type), and -re (vendre type) verbs. Grammar sources are documented in that file (multiple sources; see comments).
- **Irregular dictionary** (`convex/frenchConjugation/irregularVerbsData.json`): Generated data; do not edit by hand. See “Regenerating the irregular dictionary” below.
- **Avoiding the default load**: The rosaenlg-lib package loads `french-verbs-lefff/dist/conjugations.json` at module load time. We use **patch-package** to patch `rosaenlg-lib` so that import is stubbed to an empty object (`patches/rosaenlg-lib+4.4.0.patch`). After `npm install`, run `patch-package` (or rely on the `postinstall` script) so the patch is applied. Convex then only uses our Proxy + irregular dictionary + rule-generated tables; the 6MB JSON is never loaded.
- **Regression test** (`convex/frenchConjugation/verbConjugation.test.ts`): Compares every verb in `conjugations.json` with the output of the Proxy; run with `npm run test -- convex/frenchConjugation/verbConjugation.test.ts`. Loads the full JSON only in Node/Vitest, not in the Convex bundle.

Regenerating the irregular dictionary
-------------------------------------

Run from the project root:

    node scripts/buildIrregularVerbs.mjs

This reads `node_modules/french-verbs-lefff/dist/conjugations.json`, runs the regular verb rules for each verb, and writes any verb whose rule-generated table does not match the reference to `convex/frenchConjugation/irregularVerbsData.json`. Commit the updated JSON after running the script.

Preparation (done)
-----------------

- Find one or more sources that explain the _rules_ and _patterns_ for conjugating French verbs, not just simple lists of conjugations.

Approach (summary)
------------------

- Continue to use nlg.verbsManager.getAgreeVerb in the helper
- When the NlgLib is created, call nlg.verbsManager.setEmbeddedVerbs with a replacement verb lookup
- The new verb lookup is a Proxy that acts like the VerbsInfo interface - which is just a dictionary of verb => conjugation table (see below)
- The verb lookup uses a dictionary of irregular verbs and regular verb rules for -er, -ir and -re
- The irregular verb dictionary is generated by scripts/buildIrregularVerbs.mjs from conjugations.json
- A regression test compares every entry in conjugations.json with the output of the proxy and verifies that they match    


Conjugation table format
------------------------
- The conjugation table corresponds to the VerbInfo interface
- The VerbInfo interface is a map of code letters to an array of conjugations or participles
- The meaning of the code letters are given in this document: https://github.com/RosaeNLG/rosaenlg/blob/master/packages/french-verbs-lefff/resources/lefff-tagset-0.1.2.pdf
- The G entry has just one form of the verb - the present participle
- The W entry has just one form of the verb - the infinitive
- The K entry has the forms of the past participle agreed with masculine singular, masculine plural, feminine singular, feminine plural
- The other entries are the forms for 1st, 2nd and 3rd person, singular and plural
- The Y entry (imperative) has "NA" for forms that do not exist
- Example entry for regular verb 'parler':
`"parler": {
        "P": [ "parle", "parles", "parle", "parlons", "parlez", "parlent" ],
        "S": [ "parle", "parles", "parle", "parlions", "parliez", "parlent" ],
        "Y": [ "NA", "parle", "NA", "parlons", "parlez", "NA" ],
        "I": [ "parlais", "parlais", "parlait", "parlions", "parliez", "parlaient" ],
        "G": [ "parlant" ],
        "K": [ "parlé", "parlés", "parlée", "parlées" ],
        "J": [ "parlai", "parlas", "parla", "parlâmes", "parlâtes", "parlèrent" ],
        "T": [ "parlasse", "parlasses", "parlât", "parlassions", "parlassiez", "parlassent" ],
        "F": [ "parlerai", "parleras", "parlera", "parlerons", "parlerez", "parleront" ],
        "C": [ "parlerais", "parlerais", "parlerait", "parlerions", "parleriez", "parleraient" ],
        "W": [ "parler" ]
    }
    `
